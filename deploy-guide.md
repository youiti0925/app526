# アプリ公開の手順書（完全版）

あなたがやることは **2つだけ** です：

1. GitHubで「PR（プルリクエスト）」を作る
2. そのPRを「マージ」する

これだけでアプリが自動的に公開されます。
以下、1クリックずつ説明します。

---

## 用語の説明（先に読んでください）

| 用語 | 意味 |
|------|------|
| GitHub（ギットハブ） | プログラムのコードを保存するWebサイト。Googleドライブのコード版みたいなもの |
| PR（プルリクエスト） | 「この変更を本番に入れていいですか？」という申請書のようなもの |
| マージ | PRを承認して、変更を本番に反映すること。「OK」ボタンを押すようなもの |
| Vercel（バーセル） | コードを自動的にWebサイトとして公開してくれるサービス |
| ブランチ | コードの「作業用コピー」のこと。本番用（main）と作業用がある |

---

## 作業1：PRを作る（所要時間：1分）

### 手順1-1：ブラウザでURLを開く

下のURLをコピーして、ブラウザのアドレスバーに貼り付けて、Enterキーを押してください。

```
https://github.com/youiti0925/app526/compare/main...claude/video-to-work-standards-e4T1l
```

→ このページは「作業用のコードと本番のコードを比べる画面」です。

※ もしログイン画面が出たら、GitHubのアカウントでログインしてください。


### 手順1-2：「Create pull request」ボタンを押す

ページが開くと、上の方に **緑色のボタン** があります。
ボタンには **「Create pull request」** と書いてあります。

→ このボタンをクリックしてください。


### 手順1-3：タイトルはそのままでOK

ボタンを押すと、タイトルを入力する画面が出ます。
最初から文字が入っているので、**何も変えなくてOK** です。

もう一度 **「Create pull request」** という緑のボタンが出るので、
それをクリックしてください。

→ これでPRが作成されました！


---

## 作業2：PRをマージする（所要時間：30秒）

### 手順2-1：ページを下にスクロール

PRが作成されると、そのPRの画面が表示されます。
**ページを一番下までスクロール** してください。


### 手順2-2：「Merge pull request」ボタンを押す

ページの下の方に、**緑色の「Merge pull request」ボタン** があります。

→ このボタンをクリックしてください。


### 手順2-3：「Confirm merge」ボタンを押す

ボタンが「Confirm merge」に変わります。

→ もう一度クリックしてください。


### 手順2-4：完了を確認

**「Pull request successfully merged and closed」**
と紫色の背景で表示されれば、成功です！

→ これで作業は完了です。あとは自動的にアプリが公開されます。

---

## 作業後：アプリが公開されるのを待つ

マージが完了すると、Vercelが自動的にアプリを公開します。
**2〜3分** 待ってから、以下のURLを開いてみてください：

```
https://app526.vercel.app
```

→ アプリが表示されれば、すべて完了です！おめでとうございます！
