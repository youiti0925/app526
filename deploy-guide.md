# Vercelデプロイ手順書 - app526

## 現在の状況

コードの修正は完了しています。あとはGitHubでマージするだけでVercelに自動デプロイされます。

---

## 手順（全2ステップ）

### ステップ1：GitHubでPRを開く

1. ブラウザで以下のURLを開きます
   - https://github.com/youiti0925/app526/pulls

2. PRの一覧が表示されます
   - 「Add vercel.json...」のようなタイトルのPRをクリックします
   - ※ もしPRがなければ「New pull request」ボタンから作成します

### ステップ2：マージする

1. PRのページを下にスクロールします
2. 緑色の **「Merge pull request」** ボタンをクリックします
3. **「Confirm merge」** ボタンをクリックします
4. 「Pull request successfully merged」と表示されれば完了です

---

## マージ後の確認

1. Vercelのダッシュボードを開きます
   - https://vercel.com/ にログイン
   - 「app526」プロジェクトをクリック

2. 左メニューの **「Deployments」** をクリックします

3. 最新のデプロイのステータスを確認します
   - 🟢 **Ready** → デプロイ成功！サイトにアクセスできます
   - 🟡 **Building** → ビルド中です。2〜3分待ちます
   - 🔴 **Error** → エラーが出ています。スクリーンショットを共有してください

4. デプロイ成功後、以下のURLでアプリが表示されます
   - https://app526.vercel.app

## サイトの説明

| サイト | URL | 役割 |
|--------|-----|------|
| GitHub | https://github.com | ソースコード管理。コードの保存場所 |
| Vercel | https://vercel.com | ホスティング。コードをWebサイトとして公開する |
| アプリ | https://app526.vercel.app | 完成したWebアプリの公開URL |

### GitHubとは
- プログラムのソースコードを保存・管理するサービスです
- 「Pull Request（PR）」は、コードの変更を本番に反映する仕組みです
- PRを「マージ」すると、変更が本番のコードに取り込まれます

### Vercelとは
- Webアプリケーションを公開するためのホスティングサービスです
- GitHubと連携していて、GitHubでコードが更新されると自動的にビルド＆デプロイされます
- 「ビルド」＝ソースコードを動くWebサイトに変換する処理です
- 「デプロイ」＝ビルドした結果をインターネットに公開する処理です
